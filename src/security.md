# Security & Compliance

- All secrets must be stored in `.env` and never committed.
- Database access is restricted by credentials and least privilege.
- All user data is handled in compliance with Discord and GitHub policies.
- Automated tests and CI enforce 100% code and security coverage.
- All dependencies are kept up to date via automated housekeeping.
- Only trusted, top-tier developers have repository access.
- All integrations are reviewed for security and compliance before merging.
